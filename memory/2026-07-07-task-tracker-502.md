# 2026-07-07 task-tracker /tracker/ 502

Symptom: `http://192.168.50.109/tracker/` returned nginx `502 Bad Gateway`.

Root cause: nginx proxies `/tracker/` to `http://127.0.0.1:3000`, but `task-tracker.service` was inactive, so port 3000 refused connections.

Evidence:
- nginx error log: `connect() failed ... upstream: "http://127.0.0.1:3000/"`
- `systemctl --user status task-tracker.service`: inactive since 2026-07-06 23:21:28 CST, last exit `SIGTERM`.
- `curl http://127.0.0.1:3000/api/health`: connection refused before restart.

Action: started `task-tracker.service`.

Verification:
- `systemctl --user is-active task-tracker.service`: `active`
- `curl http://127.0.0.1:3000/api/health`: `{"status":"ok","db":true}`
- `curl http://192.168.50.109/tracker/api/health`: HTTP `200`
- `curl -I http://192.168.50.109/tracker/`: HTTP `200`

Note: source of the `SIGTERM` was not identifiable from journal logs; systemd treated it as success, so `Restart=on-failure` did not restart it.

Follow-up 2026-07-08: added `deploy/task-tracker.service`, installed it to the user systemd unit path, changed restart policy to `Restart=always`, and added `ExecReload=/bin/kill -HUP $MAINPID`.
