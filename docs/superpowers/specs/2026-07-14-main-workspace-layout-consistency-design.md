# 主工作區看板與留言操作版面一致性設計

## 目標

主工作區保留討論專用的流程限制，但在桌面與手機版都和一般工作區使用相同的四條看板河道視覺結構；同時修正留言操作按鈕在窄螢幕互相擠壓的問題。

## 看板河道

- 主工作區仍渲染 `Todo`、`Doing`、`Review`、`Done` 四條河道，並沿用一般看板的 grid 與既有 RWD breakpoint。
- `Doing`、`Review` 只作為空白的視覺河道：主工作區不在其中提供新增 task、拖移或狀態轉換控制。
- 主工作區既有的 domain 限制不變：非規則 task 只有 OWNER 能在完整收尾證據與固定期限後直接 `Todo -> Done`。
- Archived 欄位的顯示切換維持既有行為；開啟時為第五條河道。

## 留言操作

- 自己的留言操作維持編輯／儲存與刪除兩個能力，但操作容器改為垂直排列。
- 編輯（或儲存）在上排，刪除在下排；窄螢幕同樣維持此排列，按鈕可使用完整欄寬但不可互相覆蓋。
- 刪除按鈕顯示文字固定為 `刪除`；確認對話與 DELETE API 行為不變。

## 驗證

- `src/frontend.test.ts` 驗證主工作區 source 會建立 Doing 與 Review 河道，但主 OWNER 仍只會看到 `→ Done`，且沒有 `→ Doing`。
- 同一測試驗證留言刪除按鈕文字為 `刪除`，並驗證 CSS 將 `.comment-actions` 設為欄向排列。
- 以 browser preview 在桌面與 768px 以下 viewport 檢查四條河道與留言操作不重疊。
