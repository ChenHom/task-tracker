# Sim Run Amplification Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `sim/run.ts` easier to inspect and improve by saving rendered prompts, generating sprint reports, adding CI review packets, and adding simple scenario selection without adding DB schema or broad abstractions yet.

**Architecture:** Keep `sim/run.ts` as the orchestration entrypoint. Add the smallest artifact/report helpers needed around the existing flow. Store run artifacts in files under `sim-logs/<run-id>/`; defer database tables until run history or editable roster prompts are actually needed.

**Tech Stack:** TypeScript, Node.js stdlib, existing task-tracker HTTP API, Git CLI/worktrees, existing `tsx` test runner.

## Ponytail Scope

Do first:

- Save every rendered owner/member prompt as a file.
- Save shared run context files only if they are already generated by the current flow.
- Generate `report.md` and `report.json` at the end of a run.
- Expand `verifyBranches()` into review packets with changed files, diffstat, commits, and test status.
- Add simple scenario selection with an in-code `SCENARIOS` map.

Do not do yet:

- No `sim_roles`, `sim_user_profiles`, `sim_scenarios`, `sim_runs`, or `sim_prompt_artifacts` tables.
- No DB-backed prompt roster.
- No UI for scenario/profile editing.
- No tokenizer-specific accounting.
- No broad module split just to make the architecture look cleaner.

## Target Flow

```text
npm run sim -- --scenario self-directed
  -> create runId and sim-logs/<run-id>/
  -> bootstrap existing task-tracker workspace/worktrees
  -> write each rendered prompt to prompts/*.md before spawning an agent
  -> run existing member/owner sessions
  -> verify branches and write review-packets/*.md
  -> owner close prompt receives compact packet summaries
  -> write report.md and report.json
```

## File Structure

- Modify `sim/run.ts`
  Keep orchestration in place. Add run id, artifact paths, prompt writing, report writing, scenario selection, and richer branch verification.

- Modify `sim/run.test.ts`

- Optional create `sim/artifacts.ts`

## Token Efficiency Strategy

Do not compact prompts yet. First measure.

Track:

- prompt bytes per session
- total prompt bytes per run
- prompt bytes per Done task
- prompt bytes per merged branch
- zero-change sessions

Use `Buffer.byteLength(prompt, 'utf8')`. This is good enough until provider-specific token counts are needed.

## CI Review Packet Strategy

Replace current `verifyBranches()` result:

```ts
Record<string, { tsc: boolean; test: boolean; ahead: number }>
```

with:

```ts
interface BranchReviewPacket {
  branch: string;
  memberName: string;
  memberEmail: string;
  ahead: number;
  dirty: boolean;
  commits: string[];
  changedFiles: string[];
  diffstat: string;
  tsc: { status: 'pass' | 'fail' | 'skip'; outputPath: string };
  test: { status: 'pass' | 'fail' | 'skip'; outputPath: string };
  packetPath: string;
}
```

Skip conflict risk scoring for now. Changed files and diffstat are enough.

Owner close prompt should include one line per branch:

```text
- 小美 / sim/user02: tsc PASS, test PASS, 2 commits, 3 files changed, packet: sim-logs/<run-id>/review-packets/sim-user02.md
```

Keep the current terminal stats, but generate them from the same report object.

interface PromptArtifact {
  label: string;
  path: string;
  bytes: number;
}

function createRunDir(root: string, runId: string): string {
  const dir = join(root, runId);
  mkdirSync(join(dir, 'prompts'), { recursive: true });
  mkdirSync(join(dir, 'review-packets'), { recursive: true });
  return dir;
}

function writePromptArtifact(runDir: string, label: string, prompt: string): PromptArtifact {
  const safe = label.replace(/[^a-zA-Z0-9_-]+/g, '-');
  const existing = readdirSync(join(runDir, 'prompts')).filter((name) => name.endsWith('.md')).length;
  const file = `${String(existing + 1).padStart(3, '0')}-${safe}.md`;
  const path = join(runDir, 'prompts', file);
  writeFileSync(path, prompt);
  return { label, path, bytes: Buffer.byteLength(prompt, 'utf8') };
}
```

Add `readdirSync` import from `node:fs`.

function formatReviewPacket(packet: BranchReviewPacket): string {
  return [
    `# ${packet.branch}`,
    '',
    `member: ${packet.memberName} <${packet.memberEmail}>`,
    `ahead: ${packet.ahead}`,
    `tsc: ${packet.tsc.status.toUpperCase()} (${packet.tsc.outputPath})`,
    `test: ${packet.test.status.toUpperCase()} (${packet.test.outputPath})`,
    '',
    '## Commits',
    ...packet.commits.map((commit) => `- ${commit}`),
    '',
    '## Changed Files',
    ...packet.changedFiles.map((file) => `- ${file}`),
    '',
    '## Diffstat',
    '```text',
    packet.diffstat || '(empty)',
    '```',
    '',
  ].join('\n');
}
```

## Acceptance Criteria

- `npm test` passes.
- `npm run build` passes.
- Existing `npm run sim -- --smoke --scenario self-directed` still works.
- Every session writes a prompt file before spawning the agent.
- `verifyBranches()` writes review packet markdown and command output files.
- Only `pass + pass` is automatically green; `skip` requires explicit Owner review and dirty worktrees are never treated as lost work.
- Error/timeout member sessions never call the driver commit path.
- Manual runs and timers share one PID lock; a Claude quota failure cannot cancel available Codex member work.
- `npm test` type-checks both `src/` and `sim/`.
- Final run writes `report.md` and `report.json`.
- `product-ideation` can create planning tasks without touching app code.

## Deferred

- DB prompt/profile/scenario tables: add when prompt roster needs user editing or cross-run querying.
- Prompt compaction by artifact references: add after report data shows repeated prompt bytes are a real cost.
- Conflict risk scoring: add when changed files/diffstat are not enough for owner decisions.
- External repo automation beyond `--target`: add after current-repo scenario artifacts are useful.
- Tokenizer-specific token counts: add when byte counts are misleading enough to matter.
