# 產品體驗分析報告：Task Tracker UI/UX 深度體檢 (V2)

> **實作快照（2026-07-10）**：P1 專案下拉建立與 P2 workspace quick switcher 已完成。P0 欄內新增已用於 Todo/Doing/Review，但 create-task API 仍固定建立 Todo，Doing/Review 的狀態預設尚未真正生效。以下保留當時分析與決策理由。

作為資深產品經理（Product Manager），我針對當時 Task Tracker 專案的實際 UI 佈局與元件狀態進行了體驗評估。以下分析基於當時系統的結構（如底部的 6 欄位任務新增表單、4+1 欄看板等），並從**用戶體驗、操作效率、認知負載與業界最佳實踐**等維度進行剖析。

---

## 一、 產品視覺風格與核心優點 (Aesthetic & Strengths)

### 1. 獨特且高辨識度的新粗獷主義風格 (Neo-brutalism UI)
目前採用的新粗獷主義風格（如 `sketch-box` 實線黑邊框、扁平無模糊投影）能營造出大膽、潮流、極具個性的視覺氛圍，非常契合敏捷疊代與個人/小團隊的任務規劃場景。

### 2. 佈局清晰的雙欄任務詳情 (Task Detail Modal)
* **左側（內容/交流）**：聚焦於標題、描述、留言區（帶有精緻的「我」標記與時間戳記）、附件管理。
* **右側（屬性控制）**：整合狀態、優先度、指派人員與截止日期。
* **PM 評估**：典型的雙欄分流設計（類似 Linear 與 Jira），符合用戶成熟的專案管理工具心智模型，能有效將「閱讀與討論」和「屬性管理」分離。

### 3. 多功能卡片視覺階層 (Kanban Card Info Hierarchy)
* 任務卡片高度固定為 `330px`，內容超過時描述會自動以 `line-clamp: 3` (三行) 截斷。
* **PM 評估**：當卡片缺少指派人或截止日期時，前端會自動收縮，避免了多餘的空白格（Placeholder），使整個看板的縱向版面利用率最大化。

## 二、 當時 UI 畫面的優缺點剖析 (Real UI Pros & Cons)

### 1. 常駐底部的「建立新任務 (Quick Add)」表單
* **當時現狀**：表單常駐於看板底部（包含：任務名稱、描述、優先度、專案、指派人員、截止日期共 6 個輸入欄位）。
* **優點**：相較於常駐在頂部，置於底部不會在頁面剛載入時擋住看板的黃金首屏視覺（卡片列表），能維持第一眼的清爽。
* **缺點**：
  * **當卡片量大時操作不便**：一旦某個欄位的任務卡片變多、高度變長，該表單就會被擠壓到視窗底部的滾動條外。用戶必須拉到最下面才能新增任務。
  * **預設狀態不靈活**：作為一個通用表單，用戶每次新增時都必須手動選擇「專案」或「指派人」，缺乏欄位上下文（Context）感知。

### 2. 看板頂部「專案篩選」與「專案新增」並列
* **當時現狀**：看板頂部控制列將 `+ 專案` 建立輸入表單與 `專案篩選` 下拉選單並列擺放。
* **缺點**：
  * **視覺認知干擾**：過濾（Read）與建立（Write）這兩種不同性質的操作在同一個視覺區塊內平鋪，缺乏分隔，會造成使用者的認知干擾。
  * **介面擁擠**：在小螢幕（如行動裝置或窄版視窗）下，容易因為空間不足而換行跑版。

### 3. 側邊欄與全局導航切換
* **當時現狀**：用戶在進入特定工作區後，左側導航會呈現 Workspaces、Members、Search 與 Audit Log。如果需要切換工作區，必須點擊進入 Workspaces 頁面重新點選。
* **缺點**：切換動線需要經歷「點擊左下 -> 載入列表頁 -> 點選卡片」的 2-Click 流程，對頻繁多工的用戶來說效率較低。

## 三、 資深 PM 推薦之產品改良路線圖 (Actionable Roadmap)

為了在不破壞現有「新粗獷主義風格」的前提下大幅提升操作效率，建議推進以下三項產品優化：

### 1. 【P0 優先級】任務新增動線「欄位內嵌化 (Inline + Task)」
* **痛點改進**：移除底部常駐的 6 欄位大表單，改為在每個看板欄位（Todo / Doing / Review / Done）的標題下方或卡片底部，內嵌一個簡潔的 `+ 新增任務` 快速輸入框。
* **優化方案**：
  * 用戶點選 `+ 新增任務` 時，只彈出一個單行輸入框（僅需輸入標題），按下 Enter 即直接在該欄位建立任務。
  * 若用戶需要設定更詳細的描述、指派人或截止日期，可在建立後點擊卡片彈出的詳情 Modal 中原地設定，或點擊輸入框旁的「展開進階欄位」按鈕。
  * **產品價值**：極大釋放版面空間，且能根據用戶點選的欄位自動預設任務狀態（例如點擊 Done 下方的按鈕，建立的任務預設就是 Done），省去手動選擇狀態的步驟。

### 2. 【P1 優先級】將「專案新增」整合至專案選單中
* **優點改進**：將頂部導航列中長相突兀的 `+ 專案` 輸入表單移除。
* **優化方案**：在 `專案篩選` 下拉選單的最下方，新增一個帶有虛線邊框的選項 `+ 建立新專案...`。用戶點選時跳出一個新粗獷主義風格的微型對話框（Dialog）來命名專案。
* **產品價值**：保持頂部過濾區的純淨，減少不必要的常駐輸入框，版面更有呼吸感。

### 3. 【P2 優先級】側邊欄頂部整合「工作區下拉快速切換 (Quick Switcher)」
* **優點改進**：避免每次切換工作區都需要回到 workspaces 清單頁。
* **優化方案**：在側邊欄最頂部的「當前工作區名稱」旁，新增一個小下拉箭頭，點擊後原地展開工作區下拉選單，點選即可完成 1-Click 無縫切換。
* **產品價值**：縮短多工跳轉動線，對於管理多個專案群組的管理者而言是剛需功能。
